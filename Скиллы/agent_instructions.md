# agent_instructions.md

## Когда имеет смысл подключать агента

Агент полезен, когда задача требует многошаговых действий вне одного ответа:

- пройтись по сайтам и собрать материалы;
- заполнить формы;
- работать с внешними сервисами;
- обновить репозиторий;
- создать pull request;
- развернуть сайт;
- проверить сайт в браузере;
- повторять один и тот же workflow много раз;
- работать с большим числом файлов и промежуточных проверок.

## Когда агент НЕ обязателен

Агент обычно не нужен, если задача ограничивается созданием локального статического проекта по брифу и прикреплённым материалам. Такой проект можно собрать прямо в чате: проанализировать файлы, написать HTML/CSS/JS, проверить структуру и отдать ZIP.

## Хороший промпт для агента

Используй такой формат:

```text
Работай по прикреплённым файлам `web_site_skill.md`, `brief_template.md`, `design_system_template.md` и `quality_checklist.md`.

Цель: создать готовый статический сайт по брифу.

Порядок работы:
1. Изучи все файлы и архивы.
2. Составь карту материалов.
3. Собери проект в HTML/CSS/JS.
4. Проверь локальные пути, адаптив и JS.
5. Упакуй результат в ZIP.
6. В финале дай краткий отчёт: что сделано, какие допущения приняты, что можно улучшить дальше.

Не задавай вопросы, если можно принять разумное решение.
```

## Риски агентского режима

- Агент может потратить больше времени на навигацию, чем требуется для простой задачи.
- Может появиться больше промежуточных действий, которые не нужны для локального проекта.
- Если агент работает с внешними сервисами, нужно контролировать действия перед отправкой форм, публикацией или изменением данных.

## Рекомендация

Для создания набора `.md` правил агент не нужен. Для будущих задач агент имеет смысл подключать, если нужно не только сгенерировать сайт, но и, например, загрузить его в GitHub, настроить деплой, проверить в браузере, собрать референсы из интернета или внести изменения в существующий репозиторий.
