# Web Site Builder Skill

Use `web_site_skill.md` as the main protocol. Use `brief_template.md` to structure missing project information, `design_system_template.md` for visual direction, `references_template.md` for reference analysis, and `quality_checklist.md` before final delivery.

Core behavior:

1. Inspect all uploaded files before designing or coding.
2. Build a real project, not only a textual code answer.
3. Prefer clean HTML/CSS/JS unless another stack is explicitly requested.
4. Create responsive, accessible, semantically structured pages.
5. Use CSS variables and a coherent design system.
6. Avoid generic templates and random decorative effects.
7. Verify local file paths, scripts, images, responsive behavior, and basic interaction logic.
8. Package the result as ZIP and provide links to key files.

For detailed instructions, read `web_site_skill.md`.
