# Diana Art Site

**Живопись как перевод состояний.**

Это не галерея в белой комнате и не папка с картинками. Diana Art Site — маленькое авторское пространство художницы Дианы: каталог работ, дневниковые истории, архивные фрагменты, статусы наличия и прямой путь к контакту.

Сайт собран как камерная digital-галерея: чуть editorial, чуть zine, с живой типографикой, мягкими коллажами и вниманием к тому, как работа ощущается до того, как её начинают объяснять.

## Что Уже Есть

- 10 работ в каталоге, 3 из них отмечены как доступные к покупке.
- Фильтры по статусу и темам: город, поп-культура, любовь, социальное, природа.
- Два режима просмотра: компактная сетка и подробный список.
- Модальные карточки с описаниями, цитатами, тегами и медиаматериалами.
- Блок наличия, контакты Дианы и форма, которая готовит текст сообщения.
- Политика конфиденциальности для информационного сайта без backend.
- SEO, Open Graph, Twitter card, sitemap, robots.txt, favicon и JSON-LD.
- Адаптивная верстка, keyboard-friendly модалки, reduced motion и реальная Chrome/Playwright-проверка.

## Где Открывается

Релизная версия рассчитана на GitHub Pages из корня репозитория:

`https://endoflife1231.github.io/Diana-Art-Site/`

Локально:

```bash
python3 -m http.server 8000
```

Затем открыть:

`http://localhost:8000/`

## Как Устроен Проект

```text
.
├── index.html        # страница сайта, SEO, структура секций
├── css/style.css     # визуальная система, адаптивность, состояния
├── js/data.js        # каталог работ, статусы, цены, тексты
├── js/script.js      # фильтры, модалки, форма, навигация
├── assets/           # изображения, видео, favicon
├── docs/             # brief, дизайн-система, QA и история правок
├── robots.txt
├── sitemap.xml
└── .nojekyll
```

В репозитории также могут быть локальные материалы разработки: исходные фото, экспорт чата, бэкапы и Codex skills. Они не являются частью публичного сайта.

## Релизное Состояние

Старый путь `/Diana-Sitev1/` убран из релизных SEO-файлов. Видео дополнительно сжаты перед запуском: общий вес роликов уменьшен примерно с 14.4 МБ до 5.38 МБ без изменения сценариев сайта.

Проект не требует сборщика, сервера или базы данных. Это статический HTML/CSS/JS-сайт, который можно деплоить напрямую через GitHub Pages.

## Деплой На GitHub Pages

1. Запушить репозиторий на GitHub.
2. Открыть `Settings` -> `Pages`.
3. Выбрать `Deploy from a branch`.
4. Branch: `codex-update` или ветка, которую вы хотите публиковать.
5. Folder: `/root`.
6. Сохранить и дождаться публикации.

После деплоя проверить:

- главную страницу;
- мобильную сетку и список;
- открытие карточек работ;
- ссылки на Telegram/TikTok;
- `robots.txt` и `sitemap.xml`.
