# quality_checklist.md

## 1. Структура проекта

- [ ] Проект содержит все нужные файлы.
- [ ] `index.html` находится в корне проекта.
- [ ] CSS и JS подключены корректно.
- [ ] Изображения лежат в `assets/` или другой понятной папке.
- [ ] Имена файлов безопасны для web: латиница, без пробелов, без спецсимволов.
- [ ] ZIP содержит все нужные файлы.

---

## 2. HTML

- [ ] Есть `<!doctype html>`.
- [ ] Указан язык документа: `lang`.
- [ ] Есть корректный `title`.
- [ ] Есть `meta viewport`.
- [ ] Есть `meta description`, если уместно.
- [ ] Используются семантические теги.
- [ ] Заголовки идут в логичной иерархии.
- [ ] Интерактивные элементы размечены по смыслу.
- [ ] У изображений есть alt.

---

## 3. CSS

- [ ] Используются CSS-переменные.
- [ ] Есть базовый reset/normalization.
- [ ] Есть адаптивные брейкпоинты.
- [ ] Не используются лишние `!important`.
- [ ] Есть состояния `hover`.
- [ ] Есть состояния `focus-visible`.
- [ ] Цвета соответствуют брифу.
- [ ] Типографика читаемая на мобильных.
- [ ] Есть поддержка `prefers-reduced-motion`.

---

## 4. JavaScript

- [ ] JS не падает при загрузке страницы.
- [ ] Селекторы проверяются перед использованием.
- [ ] Бургер-меню работает.
- [ ] Плавный scroll работает.
- [ ] Модальные окна открываются и закрываются.
- [ ] Escape закрывает модальные окна.
- [ ] Фильтры работают.
- [ ] Поиск работает.
- [ ] Формы не отправляют данные на сервер, если backend не предусмотрен.
- [ ] Success/error уведомления отображаются корректно.
- [ ] `localStorage` используется безопасно и не ломает сайт.

---

## 5. Адаптив

Проверить ширины:

- [ ] 1440px desktop.
- [ ] 1200px laptop.
- [ ] 1024px tablet landscape.
- [ ] 768px tablet portrait.
- [ ] 430px mobile.
- [ ] 375px small mobile.

Проверить:

- [ ] Нет горизонтального скролла.
- [ ] Меню удобно на мобильных.
- [ ] Карточки не слипаются.
- [ ] Кнопки достаточно крупные.
- [ ] Текст не слишком мелкий.

---

## 6. Доступность

- [ ] Все интерактивные элементы доступны с клавиатуры.
- [ ] Видимый focus state есть.
- [ ] Контраст текста достаточный.
- [ ] Модальные окна имеют `aria-modal` или близкую доступную структуру.
- [ ] У кнопок без текста есть `aria-label`.
- [ ] Декоративные изображения имеют пустой alt или скрыты от screen readers.

---

## 7. Контент и UX

- [ ] Заголовок сайта отражает суть проекта.
- [ ] CTA понятны.
- [ ] Тексты не шаблонные.
- [ ] Секции идут в логичном порядке.
- [ ] Empty states предусмотрены.
- [ ] Статусы/цены/контакты читаются ясно.
- [ ] Нет конфликтов между брифом и дизайном.

---

## 8. Финальная проверка

- [ ] Проект открывается локально.
- [ ] Пути к изображениям рабочие.
- [ ] Нет очевидных ошибок в консоли.
- [ ] Архив скачивается и распаковывается.
- [ ] В финальном ответе есть ссылки на ZIP и основные файлы.
- [ ] В финальном ответе кратко перечислено, что сделано.
