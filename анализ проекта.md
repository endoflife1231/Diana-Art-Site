**Главные находки**
1. **Звук по hover/focus лучше переделать до деплоя.** Сейчас hero-video размьючивается при `pointerenter` и `focus`: [script.js](/workspaces/Diana-Art-Site/Diana-Sitev1/js/script.js:428). Это может не сработать из-за правил autoplay, а на клавиатуре звук при фокусе может быть неприятным. Лучше явная маленькая кнопка “звук”.
2. **SEO/social preview недособраны.** Есть `og:title/description/image`, но `og:image` относительный и нет `canonical`, `og:url`, `twitter:card`: [index.html](/workspaces/Diana-Art-Site/Diana-Sitev1/index.html:8). Для Telegram/соцсетей лучше абсолютные URL.
3. **Каталог рендерится JS-ом.** В HTML галерея пустая: [index.html](/workspaces/Diana-Art-Site/Diana-Sitev1/index.html:148), потом наполняется через JS: [script.js](/workspaces/Diana-Art-Site/Diana-Sitev1/js/script.js:113). Для MVP ок, но для SEO лучше либо статично продублировать карточки, либо добавить JSON-LD/структурированные данные.
4. **Контраст второстепенного текста местами на грани.** `--color-muted` на бумажном фоне даёт примерно `3.99:1`, ниже обычного AA-порога 4.5:1 для мелкого текста. Риск в подписях типа [style.css](/workspaces/Diana-Art-Site/Diana-Sitev1/css/style.css:222) и архивных описаниях [style.css](/workspaces/Diana-Art-Site/Diana-Sitev1/css/style.css:313).
5. **CSS накопил “слой правок поверх правок”.** Внизу есть блок `Production polish restored...`, который переопределяет более ранние правила: [style.css](/workspaces/Diana-Art-Site/Diana-Sitev1/css/style.css:511). Работает, но перед релизом я бы почистил дубли и сгруппировал стили.
6. **localStorage не валидируется.** Если в браузере останется старое/битое значение фильтра или вида, сайт может показать странное состояние: [script.js](/workspaces/Diana-Art-Site/Diana-Sitev1/js/script.js:25).
7. **Мобильное меню не меняет aria-label.** `aria-expanded` меняется, но label всегда “Открыть меню”: [index.html](/workspaces/Diana-Art-Site/Diana-Sitev1/index.html:24). Небольшой, но реальный accessibility polish.

Видео в модалках теперь останавливаются при закрытии: [script.js](/workspaces/Diana-Art-Site/Diana-Sitev1/js/script.js:218). Картинки лёгкие: около `3.5 MB`, но видео занимают около `14 MB`.

**Дизайн**
Я бы усиливал текущую идею “перевода состояний”, а не добавлял случайные эффекты. Лучшие направления: editorial/zine/raw gallery.

Что добавить:
- мягкий crossfade у блока `исходник -> результат`, можно с лёгким “печатным” смещением текста;
- явную кнопку звука на hero-video вместо hover-sound;
- FLIP-анимацию при фильтрации карточек, чтобы каталог не перескакивал резко;
- более выразительное открытие модалки: карточка будто “раскрывается” в историю;
- в архиве добавить подписи/контекст в lightbox, чтобы фото не открывались пустовато;
- один аккуратный акцентный шрифт для цитат/заголовков, например Newsreader, Playfair Display или Source Serif 4, но лучше self-host и не больше одного нового семейства.

Canva тут полезна не как часть сайта, а как фабрика ассетов: сделать короткие poster-loop видео, текстурные плашки, обложки для работ, social preview и анимированные сторис.

**Анимации**
Не советую ставить тяжёлую “вау-анимацию” везде. Для этого сайта достаточно 2-3 дорогих момента:
- hero collage: лёгкий parallax только на desktop;
- gallery filter transition;
- modal reveal;
- micro hover на карточках и контактах.

Если подключать библиотеку: GSAP/ScrollTrigger хорош для контролируемых scroll-анимаций, Rive для интерактивной маленькой иллюстрации “перевода”, Lottie для лёгких векторных декоративных вставок. Lenis я бы пока не ставил: smooth-scroll может добавить ощущение лага и спорен для доступности.

**Перед Деплоем**
Я бы не деплоил, пока не сделаны:
- чистая релизная папка: без `правки`, `backups`, экспортов и лишних исходников;
- абсолютные OG/canonical URL;
- `robots.txt`, `sitemap.xml`, favicon/preview check;
- CSP/security headers, cache headers для images/video;
- Lighthouse/PageSpeed на mobile, slow 4G, Safari iOS;
- проверка Core Web Vitals: LCP до 2.5s, INP до 200ms, CLS до 0.1;
- ручной QA: клавиатура, Escape, модалки, reduced motion, мобильное меню, контакты.

Backend не обязателен для MVP. Но для “продукта” я бы добавил минимум: CMS/админку для работ и наличия, serverless contact form или Telegram bot, простую аналитику, автооптимизацию изображений/видео, статус “зарезервировано/продано”.
